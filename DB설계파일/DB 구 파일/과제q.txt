DB와 연동해서 로그인 기능 구현
1. 로그인 성공
    - 일반유저 로그인
      그림6-43의 메세지와 함께 본인 정보 조회 및 수정기능 연결.
   - 관리자 로그인
      그림6-46의 메세지와 함께 회원 전체 정보 조회 및 수정, 삭제 기능 연결.
      전체 정보 조회에 조건 검색 기능 추가.(아이디, 이름, email, 가입일)
2. 로그인 실패
   - 로그인 실패 메세지 출력
   - 로그인 페이지 링크 표시
3. 인터페이스를 활용한 검색 기능
   - 데이터베이스에서 회원정보를 모두 조회한 후 list에 저장 
   - 아이디, 이름, email, 가입일에 대해 단독 또는
     복합적인 조건으로 검색
   - 재사용성 및 유지보수성이 높도록 작성

4. 관리자 기능 추가
   - 기존 회원의 활성화 비활성화 처리 기능
   - 회원 리스트에서 체크박스를 사용하여 체크된 회원에 대해서
     일괄 처리할 수 있도록 구현
   - 회원 리스트 화면에 회원의 활성화, 비활성화 정보 항목을 추가
   - t_member 테이블에 활성화 및 비활성화를 나타내는 칼럼 추가할 것
5. 일반 사용자 기능 추가
   - 비활성 상태의 회원으로 로그인 한 경우,
     회원 활성화후 로그인해야 하는 내용의 메세지를 출력
6. oracle procedure를 DAO에서 호출하여 회원 등록 기능 추가
   6.1 oracle package 및 procedure 작성
      - package 명 : hrm_pkg
      - procedure 명 : addMember();
        in 매개변수 : id, pwd, name, email
   out 매개변수 : 에러 메세지
      - 회원등록시 등록하려는 id가 이미 존재하면, 
        "기존에 등록된 회원이 있어, 등록할 수 없습니다." 의 에러 메시지 출력.
   6.2 DAO에 procedure 호출 기능 구현
      - CallableStatement 를 사용해야 함.
      - procedure에서 반환되는 에러메세지 처리기능 구현.
